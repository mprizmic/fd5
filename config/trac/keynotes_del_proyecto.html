Requerimientos =============== El uso de "establecimiento_edificio" se reemplaza por el uso de "ubicacion" No puede haber domicilios sin edificio. Si se crea un domicilio se lo asigna a un edificio en su creación. Cuando se crea un edificio se crea su primer domicilio. No hace falta determinar que domicilios del edificio usa cada nivel asì que se saca domicilio_localizacion El '00' en cue_anexo es el que determina que un edificio sea la sede. Sistema viejo ============= En los handler/manager se hace necesario que devuelvan un estado al controller, por eso se pasa un objeto Respuesta. En LocalizacionType se usa para 2 casos: crear la localización (incluye carga matricula). Segundo: cuando se usa para editar, en el Type se usa un EventListener para ver si el campo unidad_educativa ya existe (registro creado ==> ediciòn). En ese caso el campo se crea "disabled" en el listener del formulario. Tambieǹ se usa el factory del buildForm para crear los formulario pudiendo ponerle nombre, lo cual es ùtil para paginas con más de un formulario.
